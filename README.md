# Fix Sidecar

## **Overview**

This repository contains two Python scripts designed to process and update DICOM and BIDS-compliant JSON sidecar files:

1. **Principal Script (`dcm_convert.py`)**: Automates the conversion of DICOM files to NIfTI format using `dcm2niix`, and updates the corresponding JSON sidecar with additional metadata.
2. **Secondary Script (`update_json_sidecar.py`)**: Performs metadata extraction and updates BIDS-compliant JSON sidecar files using DICOM metadata, Exam Card files (for Philips scanners), and optional user-provided configurations.

---

## **Features**

### **Principal Script: `dcm_convert.py`**

1. **DICOM to NIfTI Conversion**:
   - Uses `dcm2niix` to convert DICOM files into NIfTI format.
   - Automatically identifies and processes JSON sidecar files generated by `dcm2niix`.

2. **Temporary Directory Management**:
   - Handles DICOM files in a temporary working directory for efficient processing.

3. **Integration with `update_json_sidecar.py`**:
   - Automatically updates JSON sidecar files with enhanced metadata using the secondary script.

### **Secondary Script: `update_json_sidecar.py`**

1. **Metadata Extraction Hierarchy**:
   - Extracts metadata (e.g., `RepetitionTime`, `PhaseEncodingSteps`, `NumberOfSlices`, `MultiBandFactor`) from JSON files as the primary source.
   - Falls back to DICOM metadata if necessary fields are missing.
   - Parses additional metadata (e.g., TR) from Exam Card files.

2. **Slice Timing Calculation**:
   - Computes `SliceTiming` using TR, slice count, multi-band factor, and slice acquisition order.

3. **Enhanced Metadata Updates**:
   - Updates fields such as `SliceTiming`, `TotalReadoutTime`, `EffectiveEchoSpacing`, and `PhaseEncodingDirection`.

4. **Custom Slice Order Support**:
   - Allows user-provided slice orders for more precise metadata customization.

## **Usage**

### **Principal Script: `dcm_convert.py`**

This script handles DICOM-to-NIfTI conversion and JSON updates.

#### **Command-Line Usage**:

```bash
python dcm_convert.py <dicom_file> <output_dir> [--exam-card <exam_card_file>] [--compute-slice-timing] [--compute-total-readout] [--slice-order <slice_order>] [--tmp-dir <tmp_dir>] [--scanner-type <scanner_type>]
```

#### **Arguments**
- `<dicom_file>`: Path to the input DICOM file or directory.
- `<output_dir>`: Directory to save the converted NIfTI and JSON files.

#### **Options**
- `--exam-card`: Path to the Exam Card file (optional).
- `--compute-slice-timing`: Enable calculation of `SliceTiming`.
- `--compute-total-readout`: Enable calculation of `TotalReadoutTime`.
- `--slice-order`: Custom slice order as a list of lists (e.g., `"[[0, 4, 8], [1, 5, 9]]"`).
- `--tmp-dir`: Optional temporary directory for processing.
- `--scanner-type`: Scanner type (default: `SIEMENS`).

---

### **Secondary Script: `update_json_sidecar.py`**

This script updates the JSON sidecar with enhanced metadata.

#### **Command-Line Usage**:

```bash
python update_json_sidecar.py <dicom_file> <json_file> <output_file> <exam_card_file> [--compute-slice-timing] [--slice-order <slice_order>]
```

#### **Arguments**
- `<dicom_file>`: Path to the input DICOM file.
- `<json_file>`: Path to the input JSON sidecar file.
- `<output_file>`: Path to save the updated JSON file.
- `<exam_card_file>`: Path to the Exam Card file for additional metadata (optional).

#### **Options**
- `--compute-slice-timing`: Enable the calculation of `SliceTiming`.
- `--slice-order`: Provide a custom slice order as a string representation of a list of lists.

## **Example Workflows**

### **Full Workflow with `dcm_convert.py`**

```bash
python dcm_convert.py example.dcm output_dir --exam-card exam_card.txt --compute-slice-timing
```

### **Custom JSON Update with `update_json_sidecar.py`**

```bash
python update_json_sidecar.py example.dcm example.json updated_example.json exam_card.txt --slice-order "[[0, 18, 36], [4, 22, 40]]"
```

## **Notes**

- Use `dcm_convert.py` for a streamlined workflow combining DICOM-to-NIfTI conversion and JSON updates.
- Use `update_json_sidecar.py` for standalone JSON file updates with detailed metadata customization.
- Include the Exam Card file for Philips scanners to provide missing parameters like `TR` or `MB Factor`.
